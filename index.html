Frame Structure

모든 화면의 기본 뼈대

Screen ├── Status Bar ← Component ├── Top Bar ← Component ├── Body ← 가변 콘텐츠 │ ├── Title + Text Area │ ├── Search Field Area │ └── Product Card Area │ └── List │ ├── Row → Module × 3 │ └── Row → Module × 3 └── CTA Bar ← Component, optional

Bar

Component가 여백 없이 영역을 채울 때.
Status Bar · Top Bar · CTA Bar

Body

내부 Area들을 Vertical Auto Layout으로 감싸는 콘텐츠 영역

Area

Component를 감싸는 여백 래퍼.
전체 너비(Full Width) 수평 슬라이스

Area Structure

Module 수에 따라 구조가 달라진다

상황 구조 예시
Module 1개 Area › Component Search Field Area › Search Field
Module 2개+, 가로 1줄 Area › List(H) › Module Product Card Area › List › Module ×3
Module 2개+, 세로 1열 Area › List(V) › Module Menu Item Area › List › Module ×4
Grid (격자) Area › List(V) › Row(H) › Module Product Card Area › List › Row › Module

Checklist

설계 전 확인하세요

화면 네이밍
①
Screen 이름은 한글로, 큰 단위부터 _로 구분했나?
②
상태(State)는 Screen 이름 마지막에 붙였나?
③
같은 기능의 화면끼리 앞부분이 통일되어 있나?
내부 구조
④
내부 Frame은 Title Case로 작성했나?
⑤
여백 없이 채우면 Bar, 래퍼는 Area?
⑥
Area 이름은 시각적 순서대로 요소를 나열했나?
⑦
콘텐츠 레이어에 API 필드명(snake_case) 붙였나?
Takeaway

이름이 곧 구조이고
구조가 곧 협업이다

Frame 이름 하나가 디자이너·개발자·AI를 같은 언어로 묶습니다.
O!Slice는 그 언어를 강제하지 않고, 쉽게 따를 수 있도록 돕습니다.
